# Day 9 Scope Expansion Policy

## Old Narrow Scope
`AI-Generated Content (AIGC) Image Detection` - originally strictly limited to detecting images generated by modern generative models (GANs, diffusion, autoregressive).

## New Broader Scope
`Synthetic Media Forensics and AI-Generated Visual Content Detection` - broadened to encompass detection, attribution, benchmarking, or forensic analysis of generated/manipulated visual media.

## Inclusion Rules
Papers are in-scope if they cover:
* AI-generated image detection
* Synthetic image detection
* GAN/diffusion/autoregressive-generated image detection
* Fake image detection
* Generated image attribution
* Image forgery / manipulation / splicing / copy-move / inpainting forgery detection
* Face forgery detection
* Deepfake image detection
* Deepfake video detection (only if the method relies on visual/forensic features and evidence can be parsed from paper text)
* Multimedia forensics focused on visual fake/generative media
* Benchmarks, datasets, or surveys in visual synthetic media forensics

## Exclusion Rules
Papers are strictly excluded if they involve:
* Audio-only deepfake or speech-only detection
* Text-only / LLM-generated text detection
* Medical diagnosis or medical image classification unrelated to forgery/synthesis (e.g. tumor, lesion, malnutrition)
* Generic object detection or segmentation with no fake/synthetic/forensic component
* Cybersecurity papers unrelated to visual media

## Why this still satisfies coherence
The topic remains highly coherent because all included papers share a core objective: evaluating whether visual media is authentic or manipulated/generated. They share common architectures, datasets (like FaceForensics++ or GenImage), evaluation metrics (AUC, EER, Accuracy), and conceptual threat models. This allows the QA engine to seamlessly compare and contrast methods across subdomains without encountering orthogonal or unrelated semantic fields.

## Why 100 papers is now achievable
By relaxing the strict boundary between "AI-generated image" and "manipulated/forged image/deepfake," we tap into a much larger and more mature pool of academic literature (especially from the rich fields of multimedia forensics and deepfake detection), easily surpassing the 100-paper threshold while still answering similar types of research questions.

## How topic drift is controlled
Topic drift is tightly controlled by strict exclusion keywords (e.g., audio, text, medical diagnosis) and requiring explicit visual/forensics-based signals. The corpus builder employs dual filtering: checking candidate keywords and rejecting abstracts that flag exclusion terms. Finally, manual validation ensures that edge cases like deepfake video are only retained if the paper focuses on parsing visual forensic evidence.
